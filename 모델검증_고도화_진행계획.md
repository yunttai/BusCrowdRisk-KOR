# 모델 검증 및 고도화 진행 계획

이 문서는 현재 레포에 남길 최종 모델 기준 검증 요약이다. 예전 수식 기반 점수 계산이나 Logistic Regression 기준 모델 설명은 제거하고, 현재 운영 artifact인 LightGBM 모델만 기준으로 정리한다.

---

# 1. 현재 기준 모델

| 항목 | 현재 값 |
| --- | --- |
| 최종 artifact | `data/risk_model_lightgbm.pkl` |
| 모델 | `LightGBM LGBMClassifier` |
| 모델 버전 | `lightgbm_full_risk_v1` |
| 예측 target | `target_no_seat_next_station` |
| target 의미 | 현재 차량이 다음 정류소에서 잔여좌석 0석이 되는지 |
| 위험 판정 threshold | `0.9` |
| 검증 방식 | 시간순 train/test split |

현재 만차확률은 사람이 만든 가중치 수식이 아니라, LightGBM의 `predict_proba` 결과다.

```text
feature row
-> encoder.transform()
-> LightGBM predict_proba()
-> fullSeatProbability
-> fullSeatRiskScore = fullSeatProbability * 100
```

test rows: 16,705건

# 3. 주요 feature

| Feature group | 주요 값 | 의미 |
| --- | --- | --- |
| `bus_state` | `remainSeatCnt`, `crowded`, `seatScarcityScore` | 현재 좌석 상태와 혼잡도 |
| `route` | `routeId`, `routeName`, `routeTypeCd` | 노선별 패턴 |
| `station_seq` | `stationSeq`, `stationSeqSegment` | 노선 안에서 현재 정류소 위치 |
| `time` | hour, weekday, weekend, holiday | 시간대/요일/공휴일 패턴 |
| `weather` | temperature, precipitation, humidity, wind_speed | 날씨 요인 |
| `event` | event count/flag | 행사 일자 요인 |

`seatScarcityScore`는 잔여좌석이 적을 때 차이를 크게 보고, 좌석이 많을 때 차이를 작게 보기 위한 로그 기반 보조 feature다.

```text
seatScarcityScore
  = 100 * (1 - (ln(remainSeatCnt + 1) / ln(seatCapacity + 1)) ^ 2.15)
```

단, 이 점수가 곧 만차확률은 아니다. 최종 확률은 LightGBM이 모든 feature를 함께 보고 계산한다.

# 4. 예측 방식

## 4.1 실시간 차량 기반 예측

실시간 도착/위치 API에서 차량 row를 만들 수 있으면 이 방식을 사용한다.

```text
GBIS 도착정보
-> 노선 필터링
-> GBIS 위치정보
-> 차량번호/차량 ID 매칭
-> 현재 잔여좌석, 혼잡도, 정류소 순번 확보
-> 외부요인 결합
-> LightGBM 예측
```

이 방식은 실제 차량 상태를 사용하므로 가장 신뢰도가 높다.

## 4.2 시간표/과거이력 기반 예측

실시간 차량 row가 없으면 `schedule_prior` proxy row를 만든다.

```text
사용자 입력 시각
-> 평일/주말/공휴일 판단
-> 노선 운행 시간표 확인
-> 같은 노선/정류소/시간대 과거 이력 검색
-> 대표 잔여좌석과 혼잡도 산출
-> 같은 LightGBM artifact에 입력
```

이 방식은 실시간 관측이 아니라 과거 패턴 기반 추정이다. 그래서 결과 JSON에는 반드시 아래 값을 함께 남긴다.

```text
predictionMode
historicalProxyGroup
historicalProxyRows
proxyRemainSeatCnt
proxyCrowded
scheduledOperating
```

# 6. 신뢰도 판단 기준

서비스나 보고서에서 확률만 단독으로 보여주면 안 된다. 특히 `schedule_prior`는 참조 이력 수준을 같이 표시해야 한다.

| 조건 | 신뢰도 |
| --- | --- |
| `predictionMode = live_vehicle` | 높음 |
| `historicalProxyGroup = route_station_hour_daytype` | schedule_prior 중 높음 |
| `historicalProxyGroup = route_station_adjacent_hour_daytype` | schedule_prior 중 비교적 높음 |
| `historicalProxyRows`가 매우 적음 | 낮음 |
| `stationName = unknown` | 사용 금지 |
| `stationSeq = 0` | 사용 금지 |
| `historicalProxyGroup = route_hour` 이하 | 낮음 |

# 7. 남은 고도화

현재 데이터 추가 없이 바로 할 수 있는 항목:

```text
1. threshold 0.8/0.9/0.95별 서비스 표시 정책 확정
2. 노선별/시간대별/정류소 순번 구간별 검증 표 최신 artifact 기준 재생성
3. schedule_prior 결과에 신뢰도 등급 표시 추가
4. 예측 JSON에서 stationName unknown 또는 stationSeq 0이면 낮은 신뢰도/사용불가로 표시
5. README에 최종 실행 명령과 예측 예시만 유지
```

추가 데이터가 있어야 좋아지는 항목:

```text
1. 장기 실시간 버스 데이터 누적
2. 정류소별 실제 승하차 수요 데이터 확보
3. 대기질/교통소통 데이터의 시간대 정확 매칭
4. 행사 데이터를 정류소 반경 기준으로 매칭
5. schedule_prior용 과거 표본 수 확대
```

# 8. 최종 보고서용 요약

```text
본 프로젝트는 경기도 버스 위치/도착 정보와 날씨, 공휴일, 행사 정보를 결합해
다음 정류소 기준 만차 위험도를 예측하는 LightGBM 기반 모델을 구축했다.

실시간 차량 feature 기반 검증에서 ROC-AUC 0.9767, F1 0.9175,
Top 10% Precision 0.9892를 기록했다.

실시간 차량 정보가 없을 때는 시간표와 과거 이력으로 proxy row를 생성하는
schedule_prior 방식을 별도로 구현했으며, 이 경우에는 예측 확률과 함께
참조 이력 수와 fallback 그룹을 같이 제공해 신뢰도를 분리 표시한다.
```
