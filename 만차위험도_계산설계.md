# 만차위험도 계산 설계

이 문서는 현재 코드 기준으로 **만차확률을 어떻게 만들고**, **어떤 데이터만 쓰며**, **예전 수식 계산을 어디까지 제거했는지**를 정리한다.

---

# 1. 현재 결론

현재 만차확률은 수식으로 직접 만들지 않는다.

```text
최종 만차확률 = LightGBM 모델의 predict_proba 결과
```

제거한 예전 방식:

```text
수식으로 직접 만든 별도 확률
수동 외부요인 가중치 보정
날씨/교통 패널티 직접 입력
별도 점수 기반 예측
```

현재 남긴 것:

```text
잔여좌석 로그 스코어 seatScarcityScore
좌석 정원 추정 estimate_capacity
혼잡도 라벨 CROWDED_LABELS
LightGBM 학습/예측 파이프라인
```

`seatScarcityScore`는 최종 확률이 아니라 모델 feature이자 화면 표시용 보조 점수다.

# 2. 예측 파이프라인

## 2.1 실시간 차량이 있는 경우

```text
GBIS 도착정보 조회
-> 해당 routeName/routeId 필터
-> GBIS 위치정보 조회
-> 차량번호 또는 vehId로 도착차량과 위치차량 매칭
-> model_hourly_features와 같은 row 생성
-> data/risk_model_lightgbm.pkl 로드
-> fullSeatProbability 출력
```

실행 예:

```powershell
busseat predict-live-risk 102000070 --route-name 5005 --output-json data/live_5005_now.json
```

## 2.2 실시간 차량이 없는 경우

실시간 도착/위치 row를 만들 수 없으면 시간표와 과거 이력으로 proxy row를 만든다.

```text
사용자 입력 시각
-> 공휴일/평일 판단
-> GBIS 노선 시간표 정보 확인
-> 해당 시간대 운행 여부 판단
-> 같은 노선/정류소/시간대 과거 이력에서 대표 잔여좌석 산출
-> scheduled_proxy row 생성
-> 같은 LightGBM artifact에 넣어 예측
```

실행 예:

```powershell
busseat predict-live-risk 102000070 --route-name 5005 --at "2026-05-25 08:00:00" --output-json data/live_5005_20260525_0800.json
```

중요:

```text
실시간 차량 예측과 시간표 proxy 예측은 같은 모델 artifact를 사용한다.
다른 알고리즘을 쓰는 게 아니라 입력 row의 출처만 다르다.
```

# 3. 모델이 예측하는 것

학습 target:

```text
target_no_seat_next_station
```

의미:

```text
현재 관측된 차량이 다음 정류소에 도착했을 때 remain_seat_cnt == 0 이 되는가
```

즉 사용자가 입력한 시점/정류소 기준으로 “해당 차량 또는 해당 시간대 proxy 차량이 다음 정류소까지 갔을 때 좌석이 없을 확률”을 예측한다.

의미:

```text

```

# 5. 현재 제외하거나 보류한 데이터

아래 데이터는 구조는 있지만 현재 충분한 원자료가 없어 최종 모델에서 제외될 수 있다.

```text
air_quality: pm10, pm25, o3, khai, air_quality_grade
traffic: avg_speed, traffic_volume, delay_time, congestion_level
expected_boardings_at_stop
```

원칙:

```text
결측 보정률이 너무 높은 feature는 모델에 넣지 않는다.
가짜 기본값으로 상관관계를 만들지 않는다.
```

# 6. 잔여좌석 로그 스코어

정량적인 잔여좌석을 이산 구간으로 자르지 않기 위해 로그 스코어를 사용한다.

```text
seatScarcityScore
  = 100 * (1 - (ln(remainSeatCnt + 1) / ln(seatCapacity + 1)) ^ 2.15)
```

기본 좌석정원 45석 기준 예:

```text
잔여좌석 0석  -> 100점
잔여좌석 1석  -> 97점
잔여좌석 2석  -> 93점
잔여좌석 3석  -> 89점
잔여좌석 10석 -> 63점
잔여좌석 11석 -> 61점
```

해석:

```text
잔여좌석이 적을 때는 1석 차이를 크게 본다.
잔여좌석이 많을 때는 1석 차이를 작게 본다.
이 값은 확률이 아니라 모델 입력 feature다.
```

# 7. 출력값 해석

예측 JSON의 핵심 필드:

```text
fullSeatProbability: 모델이 예측한 만차확률
fullSeatRiskScore: fullSeatProbability * 100
riskLevel: low / medium / high
predictionMode: live_vehicle 또는 schedule_prior
featureSource: live_gbis 또는 schedule_historical_proxy
```

예:

```json
{
  "routeName": "5005",
  "stationName": "순천향대학병원",
  "predictionMode": "schedule_prior",
  "fullSeatProbability": 0.1715,
  "fullSeatRiskScore": 17.15,
  "riskLevel": "low"
}
```

# 8. 현재 필요한 명령

학습 데이터 생성:

```powershell
busseat build-model-hourly-features --export-csv data/model_hourly_features.csv
busseat build-target-labels
busseat export-training-dataset --export-csv data/training_dataset.csv
```

모델 진단:

```powershell
busseat diagnose-model --model lightgbm --output-json data/model_diagnostics.json --output-md data/model_diagnostics.md
```

최종 artifact 학습:

```powershell
busseat train-lightgbm-artifact --output-artifact data/risk_model_lightgbm.pkl --output-json data/risk_model_lightgbm_artifact.json
```

저장 row 예측:

```powershell
busseat predict-feature-risk --artifact data/risk_model_lightgbm.pkl --route-name 5005 --station-name 순천향대학병원 --at "2026-05-25 04:03:33"
```

실시간/시간표 proxy 예측:

```powershell
busseat predict-live-risk 102000070 --route-name 5005 --at "2026-05-25 08:00:00"
```

# 9. 코드 기준

현재 유지해야 하는 핵심 파일:

```text
src/busseat_ai/modeling.py
src/busseat_ai/external_context.py
src/busseat_ai/targets.py
src/busseat_ai/cli.py
src/busseat_ai/api.py
src/busseat_ai/services/risk.py
```

`services/risk.py`는 더 이상 최종 확률을 계산하지 않는다.

```text
남은 역할:
1. 혼잡도 라벨
2. 노선유형별 기본 좌석정원
3. 잔여좌석 로그 스코어
```
