# 레포 업로드 체크리스트

이 문서는 GitHub 같은 공개/공유 레포에 올리기 전에 확인할 항목을 정리한 것이다.

---

# 1. 올리면 안 되는 파일

아래 파일은 공개 레포에 올리지 않는다.

```text
.env
sureing_occupancy_backup.sql
data/busseat.db
data/busseat.before_*.db
data/*.pkl
data/*training*.csv
data/*features*.csv
data/live_*.json
data/hourly_risk_*.csv
data/hourly_risk_*.json
```

이유:

```text
.env에는 API 키가 들어간다.
SQL dump와 DB에는 원본 수집 데이터가 들어간다.
모델 pkl과 대형 CSV는 용량이 크고 재생성 가능한 산출물이다.
```

위 항목은 `.gitignore`에 반영했다.

---

# 2. 올려도 되는 핵심 파일

레포에는 아래 중심으로 올린다.

```text
src/
tests/
scripts/
pyproject.toml
requirements.txt
.env.example
README.md
*.md 보고서 문서
example-db/README.md
```

`example-db`의 샘플 CSV는 데모 목적이면 올려도 되지만, 실제 제출 레포 용량을 줄이고 싶으면 `scripts/build_example_db.py`로 재생성하는 방식만 안내해도 된다.

---

# 3. 업로드 전 실행할 명령

## 3.1 테스트

```powershell
$env:PYTHONPATH='src'
python -B -m unittest discover -s tests
```

## 3.2 민감정보 검색

```powershell
rg -n "PUBLIC_DATA_SERVICE_KEY=.+|ITS_API_KEY=.+|DATA_GG_SERVICE_KEY=.+|serviceKey=|apiKey=|KEY=" -S . -g "!data/**" -g "!sureing_occupancy_backup.sql" -g "!**/__pycache__/**"
```

검색 결과에 실제 키 값이 나오면 커밋하면 안 된다.

## 3.3 git 초기화 후 확인

```powershell
git init
git status --short
```

`git status`에 아래 파일이 보이면 안 된다.

```text
.env
sureing_occupancy_backup.sql
data/busseat.db
data/*.pkl
data/*.csv
```

# 5. 추천 첫 커밋 범위

첫 커밋은 아래 정도가 적당하다.

```powershell
git add .gitignore .env.example README.md pyproject.toml requirements.txt src tests scripts *.md data/.gitkeep
git status --short
git commit -m "Initial bus full-seat risk prediction pipeline"
```

커밋 전에 `git status --short`를 반드시 확인한다.
